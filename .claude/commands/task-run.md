ステアリングファイルに基づきタスクを実装するコマンド。

## 引数

$ARGUMENTS: ステアリングファイルのパス（例: docs/steering/3-quiz-component.md）

## 実行手順

1. 指定されたステアリングファイルを読み取る
2. テスト設計セクションに基づき、テストコードを先に作成する
3. テストが通る実装コードを作成する
4. `docs/TODO.md` の該当タスクのステータスを更新する

## 注意事項

- スペック駆動開発: 必ずテストを先に書き、その後に実装する
- テスト実行やビルド確認など出力が多い処理は、Taskツールでサブエージェントに委譲すること
- ステアリングファイルの実装方針・影響範囲に従って作業すること
- 影響範囲外のファイルは変更しない
- 実装中に方針の変更が必要な場合は、ユーザーに確認すること
