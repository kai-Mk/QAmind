応用フェーズ（Practice 8〜10）の学習者コードを添削するコマンド。

## 引数

$ARGUMENTS: プラクティス番号（例: 8）

## 実行手順

1. `data/progress.json` を読み取り、対象プラクティスのレビュー回数を確認する
2. レビュー回数が3回に達している場合は、レビュー終了済みであることを伝える
3. `practices/practice$ARGUMENTS/` 内のテストコードと実装コードを読み取る
4. `practices/practice$ARGUMENTS/TEXT.md` の要件を読み取る
5. 以下の観点でレビューする:
   - テスト観点の網羅性（正常系・異常系・境界値）
   - 単体テストと結合テストの役割分担の適切さ
   - テストの可読性・保守性
   - 要件に対する漏れの有無
6. Taskツールのサブエージェントでvitestを実行し、テストが通ることを確認する
7. レビュー結果を報告する:
   - OK → `data/progress.json` のstatusを `completed` に更新、`completedAt` を記録
   - NG → `data/progress.json` の `reviewCount` をインクリメント、指摘内容を `data/mistakes.md` に追記
   - 3回目のレビュー → 結果に関わらず `completed` に更新

## レビュー結果の報告形式

### 良い点

- （具体的に何が良かったか）

### 改善点

- （具体的に何をどう直すべきか、コード例を含める）

## 注意事項

- 応用フェーズ（Practice 8〜10）のみ対応する。それ以外の番号が指定された場合はエラーを出す
- テスト実行はTaskツールでサブエージェントに委譲すること
- 指摘は具体的に。コード例を添えて改善方法を示すこと
- mistakes.mdへの追記はMarkdown形式で、コードブロックを含めること
