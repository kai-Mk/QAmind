# アプリ概要

これからAIエージェントを用いたアプリ開発かデフォルトになる中で、ジュニアエンジニアのステップアップの在り方が変わってくると考えられる。
いままでのエンジニアのステップアップとしては、テストや保守・運用を経験して、実装、設計など下流⇒上流と経験していった。
AIエージェントによってテストや実装は、置き換えられる。
そこでエンジニアに必須となる知識はより設計に意識を向ける必要がある。

そこでジュニアエンジニアが「品質思考」を身に着けるための学習アプリを作成する。
テスト設計に絞った学習アプリとし、単にコードを書くのではなく品質を意識した何を守るべきかの判断をできる力を身に着ける。

## 提供価値

- 要件からテスト設計を考える力
- 単体テストと結合テストを適切に使い分ける
- ＡＩが書いたコードの品質を見抜ける

# 対象ユーザー

- 実務1~3年のジュニアエンジニア
- テストコード経験の浅い人。書いたことがない人
- TDD未経験者
- AIエージェントを使い始めた人
- テスト設計、品質設計に自信がない人

# 学習構造

インプットフェーズ

- なぜテストをする必要があるのかをテキストベースでよむ
- 単体テスト設計と実際のコーディング。マジで簡単な関数からどのようにテストを書くか
- 結合テストパターン

応用フェーズ

- 要件に対してテストを設計してTDDで開発の経験をする
  守るべき事故を特定

単体と結合の役割分担を設計

テスト観点を整理

テストコードを書く

# 実行環境

- vitestかjestを使ったjsベースのテスト環境
- IDE環境でできるようにターミナルベースのアプリ。InkというCLIツールを使用する
- DBは用意せず、プロジェクトに進捗管理用のファイルを置いておく
- 理想はclaude codeで学習をし各カリキュラムの添削と進捗管理をしてもらう。

# 実際の動作

- `test start`みたいなコマンドを打ったら、学習が開始される。
  → /practice1/○○.tsでテストの学習をしてください。見たな感じでスタート
- `test review`みたいなコマンドで実際に手を動かした内容の添削を行う。レビューは3回までとし、3回終わったら進捗管理ファイルを更新する
- `test start #6`みたなコマンドで、途中からスタートできるようにする。
